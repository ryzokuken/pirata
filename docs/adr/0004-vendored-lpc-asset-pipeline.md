# 0004 — Vendored LPC asset pipeline

**Status:** Accepted (2026-07-12) · **Milestone:** Asset overhaul

## Context

Through v0 (M0–M4) the game rendered procedural placeholder tiles and rectangles
across both maps. The spec (§4.6) commits to LPC art with CI-enforced
attribution, without slowing the game or the build down.

## Decision

Assets are generated by committed scripts and vendored into the base pack:
`build-tileset.ts` packs curated tiles from the pinned LPC base-assets zip
(SHA-256 verified); `compose-characters.ts` composites per-layer walk sheets from
the Universal LPC generator repo at a pinned commit, recoloring by palette-ramp
substitution, emitting per-sheet credits files from `sheet_definitions`, and
re-encoding each sheet as indexed PNG8 (pixel-identical, ~8× smaller). Maps
carry a terrain theme (coastal town, cave cove) resolved by `map-defs.ts` at
generation time. The pack manifest declares its art in a Zod-validated `assets`
section; core stays asset-blind (sprite lookups live in content and client).
`check-attribution.ts` enforces both the attribution table and a 600KB total
budget.

Rejected: build-time atlas packing (tooling cost, negligible win at this scale)
and runtime CDN fetches (offline dev breaks, attribution bypassed, supply-chain
risk).

## Consequences

- Regenerating art is `pnpm build:tileset` / `pnpm build:characters`; committed
  PNGs are build products of pinned inputs, never hand-edited.
- Third-party packs declare art the same way; loader gaps are loader bugs.
- The walkability golden tests pin collision across visual map rebuilds.
- Hollow 1-tile-thick buildings render with uniform brick walls; a facade/roof
  split needs multi-tile structures and is deferred.
